# After-task: ship profile-likelihood intervals under a random effect (v0.2)

## Task goal

Make profile-likelihood `confint`/`profile` available for the fixed-effect
coordinates of a random-effects fit (`CTmax ~ <fixed> + (1 | group)`). The
underlying machinery already worked but was gated: `tls_profile_nll_fun()` passes
`random = inputs$random`, so each profile point re-runs the Laplace, but
`profile()` aborted on any RE fit and `confint()` routed RE fits to Wald. Second
of the maintainer-approved ("do all") v0.2 items; chosen now because it is R-only
(no engine change, no byte-identical risk) while the bayesTLS-definition research
for T_crit / heat-injury ran in the background.

## What was implemented

- **`R/profile.R`**: removed the hard "RE not implemented (Phase 2)" abort so
  coordinate targets profile under the RE via the existing Laplace re-run. Added a
  guard so a **group contrast** under an RE errors clearly (the contrast refit
  recodes the design and drops the random block -- it would be silently wrong).
- **`R/confint.R`**: replaced the "RE -> Wald for everything" routing with
  selective routing in a new internal `tls_confint_profile_re()`:
  - fixed-effect coordinates profile (Laplace re-run at each grid point);
  - `sigma_CTmax` stays on its log-scale Wald interval (no profile coordinate
    yet);
  - a non-closing profile (or non-pd Hessian) falls back to **Wald**, not the
    parametric bootstrap, which does not carry the random block;
  - `method = "bootstrap"` on an RE fit returns Wald with a message.
  The non-RE path is textually unchanged (the RE logic is dispatched behind
  `if (!is.null(object$re))`), so no fixed-effects behaviour moved.
- **`R/plotting.R`**: `plot_confidence_eye()` forces Wald for an RE fit (with a
  message) so the eye stays a fast visual rather than re-running the Laplace for
  every parameter at every grid point.

## Mathematical contract touched

No new model and no engine change. The profile under the RE is the standard
profile of the Laplace-approximate marginal likelihood: at each grid value of the
target coordinate, the random effects `b_CT` are re-integrated (Laplace) and the
remaining fixed effects re-optimised, exactly as `tls_profile_nll_fun()` already
does with `random = "b_CT"`. `sigma_CTmax` is intentionally not profiled (there is
no `log_sd_CT` profile target); it keeps the documented log-scale Wald interval.

## Tests of the tests

The profiling assertion is `skip_on_cran` with a tiny config (5 temps x 4 times x
2 reps, 6 RE groups, `npoints = 6`) so it runs locally / on CI-non-CRAN but not on
CRAN. The routing assertions (`sigma_CTmax` -> Wald, bootstrap -> Wald, contrast
error) run a no-profile path, so they are fast and always run. The
contrast-under-RE error reuses the existing 2-group + RE fit (no extra fit).

## Consistency audit

The RE interval story now agrees across `confint.R`, `profile.R`, `plotting.R`,
`docs/design/08`, the capability matrix, known-limitations, NEWS, and ROADMAP:
profile is available for the fixed effects under the RE; `sigma_CTmax`, the
parametric bootstrap, and the Confidence Eye stay on Wald; an RE-aware bootstrap
is the remaining RE item.

## What did not go smoothly

- Nothing material. The change is well isolated because the slow inner machinery
  already existed and was correct (overnight-session finding); this slice was
  mostly removing a gate and adding selective routing + the right fallback.
- Caught during design: a naive "remove the abort" would have let group-contrast
  profiling proceed into a refit that drops the random block (silently wrong) --
  hence the explicit contrast-under-RE guard.

## Known limitations and next actions

- Profiling under an RE is slow (each grid point is a nested Laplace fit); it is
  opt-in via `method = "profile"` and the Confidence Eye stays on Wald. No
  warm-start / point-reduction optimisation was added beyond the existing MLE
  warm start.
- `sigma_CTmax` still has only a Wald interval; a profile or an RE-aware bootstrap
  for the variance component is the remaining RE item.
- Next v0.2 items (this autonomous run): T_crit (definition confirmed:
  `T_crit = CTmax + z*log10(r*/100)`, a deterministic transform with a profile/delta
  CI -- do not copy bayesTLS's `runif` rate draw), then heat-injury prediction
  (forward-Euler dose accumulation + optional Sharpe-Schoolfield repair), then the
  covariates-on-shape engine refactor (byte-identical gate), then polish.
