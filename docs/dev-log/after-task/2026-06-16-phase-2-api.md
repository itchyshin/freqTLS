## Outcomes

- The S3 surface reads well: `print` shows the call, family, tref, a one-line
  data summary, the natural-scale estimates table, the optimiser/convergence
  state, and logLik/df/AIC. `summary` adds Wald z and p-value columns.
- `tidy_parameters()` produces the documented 8-column tibble; the Wald CIs are
  built on the internal link scale and back-transformed, so they respect bounds
  (`z > 0`, `low in (0,1)`) and are asymmetric on the natural scale where the
  link is nonlinear, symmetric for identity-link `CTmax`.
- Equivariance verified: the `z` Wald CI equals `exp()` of the internal
  `beta_logz` Wald CI to a max absolute difference of 0. This is the property
  Phase 3's profile machinery relies on.
- Grouped fits extract per-group `CTmax:A/B` and `z:A/B` correctly, indexing the
  repeated `beta_CT`/`beta_logz` rows of `summary(sdreport, select="fixed")`
  positionally (a name-based lookup would have collided).
- Model selection works: on overdispersed data (phi=8) the beta-binomial beats
  the binomial on both logLik (-142.9 vs -192.7) and AIC (297.7 vs 395.4); on
  clean binomial data the beta-binomial collapses to near-binomial (phi -> ~4.4e5)
  and AIC prefers the simpler binomial by 2.
- `simulate_tls()` misuse now errors clearly (see below), closing the footgun.

- SPEC deviation flagged (minor, documented): `tidy_parameters()` builds Wald

  on the same scale). `up` still uses the natural-scale delta interval from the
  ADREPORTed `up` SE, as the spec's profile table anticipates for `up`.

## Tests Of The Tests

- Recovery thresholds are absolute where the spec is absolute (`expect_lt(abs(.))`)
  rather than relative, after finding this waldo version rejects
  `expect_equal(scale = 1)`; the binomial CTmax/z/low/up checks would otherwise
  have used a misleading relative tolerance on `low ~ 0.02`.
- The bb-beats-binom test uses phi=8 (strong overdispersion) so the logLik/AIC
  gap is ~50/~98 -- far from the decision boundary, so it is not seed-fragile.
- The near-binomial test asserts phi > 1000 (observed ~4.4e5) and a logLik gap
  < 0.01, both with comfortable margins.
- The print test checks the cli header via `expect_message` (cli writes to the
  message stream) and the estimates table via `expect_output` (stdout) -- a
  single `expect_output("profileTLS")` would have silently missed the header.
- The misuse tests assert on the error text (`atomic vector`,
  `one value per group`, `single value for an ungrouped`) so a future change
  that weakens the guard fails loudly.

## What Did Not Go Smoothly

- First `tidy_parameters()` run errored ("missing value where TRUE/FALSE
  needed"): the occurrence counter for repeated internal names used a named
  integer vector whose first lookup returned `NA` (not `NULL`), so `%||%` did not
  catch it. Switched to a `list()` accumulator.
- The grouped/mismatch error messages first tripped cli's "Multiple quantities
  for pluralization" because a single message string referenced two quantities
  (`ng` and `len`) around a `{?s}` marker. Pinned the pluralization quantity with
  `cli::qty(ng)`.
- `expect_output(print(fit), "profileTLS")` failed because cli `cli_text()`
  writes to the message connection, not stdout; the stdout capture only sees the
  printed data frame. Split into `expect_message` + `expect_output`.

## Team Learning

- When extracting SEs for grouped coordinates from a TMB sdreport summary, index
  positionally: `summary(sdr, select="fixed")` repeats row names like `beta_CT`
  once per group, so a name lookup collapses them. `tls_wald_natural()` records
  the pattern for Phase 3 to reuse.
- cli message text goes to stderr/message, not stdout; test cli headers with
  `expect_message`, not `expect_output`. And keep one pluralized quantity per cli
  string (or pin it with `cli::qty()`).
- Building Wald intervals on the link scale and back-transforming gives
  bound-respecting, equivariant intervals "for free" and puts Wald and profile on
  the same footing for Phase 3.

## Known Limitations

- Intervals in this phase are Wald only (`interval_type = "wald"`). Profile
  intervals, the 12 identifiability warnings, and the non-closing-profile
  handling arrive in Phase 3; until then `tidy_parameters()` does not warn on
  weakly identified targets.
- `up`'s Wald interval is a symmetric delta interval on the natural scale and can
  in principle exceed 1 at its upper endpoint when `up` is near the boundary
  (observed 1.005 for one beta-binomial fit); this is expected for a delta
  interval and is one motivation for the Phase-3 native re-rooting of `up`.
- `coef.profile_tls()` returns non-unique names for grouped fits (e.g.
  `CTmax:A`, `CTmax:B` are unique, but the bare named vector is keyed on the
  `parameter` column); use `complete = TRUE` for the disambiguated data frame.
