# After-task: flagship grouped-shape README example

## Task goal

Add a worked front-page example of a v0.2 capability without one. The random
intercept already had a README section; the distinctive un-exampled capability
was grouped covariate effects on the shape parameters (`low` / `up` / `log_k`).
Item 2 of the remaining roadmap. Docs-only (no package code change).

## What was added

A new README section, *Population differences in curve shape*: a heat-tolerant and
a heat-sensitive population differ in both `CTmax` and the curve steepness `k`
(`simulate_tls(group = c("tolerant", "sensitive"), CTmax = c(38, 35), k = c(8,
3), ...)`), fitted with grouped `CTmax` / `log_z` / `low` / `up` / `log_k`. The
example prints the per-group steepness with profile-likelihood compatibility
intervals and faceted survival curves.

Verified output (real, from `build_readme()`): `k:tolerant` 8.86 [7.43, 10.6],
`k:sensitive` 2.90 [2.40, 3.48], both `conf.status = ok` -- non-overlapping
intervals on a shape parameter, so the shape difference is identified, not
assumed. The faceted figure shows steep survival cliffs in the tolerant panel and
gradual declines in the sensitive panel, observed proportions overlaid.

This example depends on the formula-group-vector fix committed just before it
(`4264609`): `plot_survival_curves()` previously errored on a formula-grouped fit.
