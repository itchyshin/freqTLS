# After-task: vendor snowgum_psii (Beta-family real dataset)

## Task goal

Item 3b: vendor a new openly-licensed dataset so the v0.2 **beta** family has a
real data showcase (it previously had none). Maintainer chose `snowgum_psii` from
the `landscape_scout` candidates. CC BY 4.0; vendored with attribution.

## Provenance + licence (the human-gate this slice was waiting on)

`snowgum_psii` is vendored from the `bayesTLS` package
(`inst/extdata/data_function_PSII_TDT_snowgum.csv`), redistributed under **CC BY
4.0** (the bayesTLS package licence, confirmed in its `DESCRIPTION`). Snowgum
*Eucalyptus pauciflora* photosystem-II thermal tolerance: Fv/Fm measured before
and after a temperature x duration exposure. The response is the retained PSII
proportion `prop = final_fvfm / initial_fvfm` in `[0, 1]` -- the natural input to
the beta family. Attribution added to `R/data.R`, `inst/CITATION`,
`inst/COPYRIGHTS`, and the README, exactly as for shrimp / zebrafish.

## Data shape and a real wrinkle

319 rows; temp in {28, 34, 39, 42, 45, 48} C; duration in {5, 15, 30, 60, 120}
**minutes** (not hours -- documented; set `tref` on the minute scale). **60 of
319 rows have `final_fvfm == 0`** (complete PSII loss), so `prop` sits exactly at
the `0` boundary. The beta density is undefined at 0/1, so `fit_tls(family =
"beta")` clamps those inward (to `1e-6`) with a warning. Decision: **vendor the
raw proportion unchanged** (zeros included) so the complete-loss observations are
visible, not hidden by pre-clamping; the clamp + warning is the honest,
documented behaviour. Verified fit: CTmax 46.5 C [45.9, 47.1], z 6.5 [5.9, 7.2],
conv 0, pdHess TRUE -- biologically sensible for Eucalyptus PSII.
