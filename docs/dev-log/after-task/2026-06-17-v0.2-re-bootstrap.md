# After-task: RE-aware parametric bootstrap for sigma_CTmax (v0.2)

## Task goal

Give a random-effects fit a prior-free interval for the variance component
`sigma_CTmax` (and the fixed effects). The Wald interval was the package's
weakest -- `sigma_CTmax` is a maximum-likelihood variance, biased low and
boundary-prone -- so a prior-free bootstrap is the most valuable of the deferred
items. First of the post-v0.2 deferred items (maintainer "go ahead").

## What was implemented

`R/bootstrap.R`: removed the RE-abort guard and added an RE-aware path to
`tls_bootstrap_replicates()`:

- **Draw**: per replicate, redraw the group deviations `b_g ~ N(0, sigma_hat)`,
  inject them into the compiled model's full parameter vector
  (`fit$obj$env$last.par.best`), and recompute `p` via `obj$report()` -- the cpp
  forward map, no R duplication. Then draw `y` from `p` (the shared
  `draw_from_p()` helper; binomial / beta-binomial / beta).
- **Refit**: `MakeADFun(..., random = "b_CT")`, optimise, re-pin `obj$fn(opt$par)`
  before `report()` (re-solves the inner Laplace, as `tls_estimates()` does), and
  collect `sigma_CTmax` alongside the fixed effects.
- `rep0` now uses the full par for an RE fit (the optimiser par is fixed-only).

`R/bootstrap.R` (`tls_boot_target`): `sigma_CTmax` percentiles taken on the log
scale (bound-respecting, consistent with its Wald interval).
`R/confint.R`: the RE + `method = "bootstrap"` branch now proceeds to the
RE-aware bootstrap (with a one-line speed note) instead of routing to Wald.

The no-RE bootstrap path is byte-identical (the new logic is guarded on
`is_re`; `draw_from_p(p_hat)` reproduces the old `draw_y`).

## Checks run and exact outcomes

- `devtools::test()` -> `PASS 383 | FAIL 0 | WARN 0 | SKIP 0` (was 379; the RE
  test file gained the RE-bootstrap recovery test and the stale "bootstrap ->
  Wald / engine errors" assertions were updated to the new contract). Existing
  non-RE bootstrap tests pass unchanged (byte-identical).
- `rcmdcheck` (`_R_CHECK_FORCE_SUGGESTS_=false`) -> pending in the check-log.
- Behaviour: an 8-group RE fit, `confint("sigma_CTmax", method = "bootstrap",
  nboot = 40)` returns a finite positive interval with `method = "bootstrap"`;
  the `CTmax` bootstrap covers the truth.

## Tests of the tests

The recovery test is `skip_on_cran` (a Laplace refit per replicate is slow) with
a small config and `nboot = 40`, deterministic `boot_seed`. It asserts a finite
positive ordered `sigma_CTmax` interval and `CTmax` truth coverage, both with
`method = "bootstrap"`. The contract test that previously asserted RE bootstrap
-> Wald (and that the engine errors on RE) was rewritten; the still-true part
(`sigma_CTmax` stays Wald under `method = "profile"`) was kept.

## What did not go smoothly

- First run failed at `obj$report(fit$par)`: for an RE fit the optimiser par is
  fixed-only, so the report needs the full par vector
  (`fit$obj$env$last.par.best`). Fixed.

## Known limitations carried forward

- Slow (a Laplace refit per replicate); use a small `nboot`.
- Random effects beyond a single `CTmax` intercept (random slopes, REs on other
  sub-parameters, crossed factors) remain out of scope.
