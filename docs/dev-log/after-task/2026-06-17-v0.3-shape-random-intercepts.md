# After Task: Random intercepts on the shape coordinates low and log_k (v0.3)

**Date:** 2026-06-17

**Task:** Item-5 stretch — extend the random-intercept system from `CTmax` /
`log_z` to the shape coordinates `low` and `log_k`
(`<param> ~ <fixed> + (1 | group)`), completing "random effects on any
sub-parameter." The upper asymptote `up` is excluded (its nested gap has no single
internal coordinate). Crossed / nested REs are deferred (a stacked-random-vector
engine redesign — out of scope for a safe incremental slice).

## Created / Changed

- **Engine** `src/profile_tls.cpp`: added `re_index_low` / `re_index_logk`
  (DATA_IVECTOR) and `b_low` / `log_sd_low` / `b_logk` / `log_sd_logk` (PARAMETER,
  appended at the end). In the loop the deviation is added on the shape's internal
  scale (logit for `low`, log for `k`) under a `b_*.size() > 0` guard; up_i is
  recomputed from the shifted `low_i` (it can no longer use the `up_s` fast path
  when `low` has a RE). The no-RE path is the original code verbatim → byte-identical.
- **Parser** `R/formula.R`: `low` and `log_k` shape RHSs route through
  `tls_extract_re()` (the RE bar is stripped, the fixed design built from the fixed
  part); a bar on `up` is rejected. The same-grouping warning is generalised to
  fire once per shared grouping across CTmax / log_z / low / log_k.
- **Helpers / plumbing**: `tls_re_blocks()` and `tls_has_re()` (`R/utils.R`) gain
  the `low` / `log_k` blocks; `fit_tls()` builds both blocks (data / params / map /
  random / `fit$re_low` / `fit$re_logk` / `sigma_*` rows); `tls_contrast_refit()`
  carries the new engine fields; `extract.R`, `confint.R`, `bootstrap.R` extend the
  variance-component Wald / routing / redraw to `sigma_low` / `sigma_logk`;
  `print` iterates `tls_re_blocks()`; `ranef()` already iterated the blocks.
- **Simulation** `R/simulate.R`: `re_sd_low` / `re_sd_logk` modes (deviation on
  `logit(low)` / `log(k)`), combinable with `re_sd` / `re_sd_z`; RNG-stream- and
  bit-preserving for calls that omit them.
- **Tests** `tests/testthat/test-shape-random-effects.R` (new); the two scope tests
  in `test-formula.R` / `test-random-effects.R` updated (a `low` RE is now accepted;
  only an `up` RE is rejected).
- **Docs synced:** `NEWS.md`, `docs/design/08-random-effects.md`,
  `docs/design/46-capability-matrix.md`, `docs/dev-log/known-limitations.md`,
  `docs/dev-log/decisions.md`, `README.Rmd` (re-knit `README.md`), regenerated
  `man/` (`simulate_tls.Rd`, `tls_bf.Rd`; also corrected a stale cross-package
  `\link` in `heat_injury_envelope.Rd` that pointed at `bayesTLS::plot_heat_injury`).

## Outcomes

Byte-identical held throughout (the guard pattern keeps the no-RE shape path the
original code verbatim). The shape REs recover their fixed effects and surface
positive `sigma_low` / `sigma_logk`; they reuse the `tls_re_blocks()` path, so
ranef / sigma rows / profile / bootstrap needed no per-shape special-casing.

## Consistency Review

RE scope is now stated uniformly as "CTmax / log_z / low / log_k, not up" across
design/08, known-limitations, capability-matrix, NEWS, README, decisions, and the
parser error message. The independent-variance warning and the per-coordinate
sigma-scale notes (°C / log z / logit low / log k) are consistent.

## Tests Of The Tests

The no-RE test asserts the formula fit equals the column fit (1e-8) and that
`sigma_low` / `sigma_logk` are absent (the R-level byte-identical guard). The
generalised same-grouping test asserts both the warning AND a successful two-block
fit. Recovery assertions are deliberately lenient (shape REs are weakly identified):
convergence + a finite positive sigma + CTmax/z recovery, not a tight sigma band.

## What Did Not Go Smoothly

Two stale scope assertions (written in the log_z slice, asserting a `low` RE is
rejected) failed once `low` REs became valid — updated to assert `up` is the only
rejected shape RE. Caught and fixed before commit. The re-`document()` also
surfaced (and fixed) a stale cross-package `\link` in `heat_injury_envelope.Rd`.

## Team Learning

The `tls_re_blocks()` abstraction (introduced at N=2 in the log_z slice) paid off:
adding two more RE coordinates needed no changes to ranef / bootstrap / profile
routing. The byte-identical guard pattern (keep the no-RE branch as the original
expression verbatim, gate the RE addition on `b_*.size() > 0`) made an intricate
shape engine safe to extend.

## Known Limitations

Shape REs are more weakly identified than `CTmax` (need data informing the
asymptote / steepness per group). `sigma_low` / `sigma_logk` are ML, biased low
with few groups, on internal (logit / log) scales. `up` has no RE. Crossed / nested
/ correlated REs remain out of scope (a stacked-random-vector engine redesign;
`bayesTLS` for correlated structures).

## Next Best Task

log_k). The genuinely remaining RE work — crossed / nested grouping factors and
correlated multivariate random effects — is a larger engine-design project (a
