# After-task: absolute-target heat injury (`target_surv`)

## Task goal

Generalise `predict_heat_injury()` from the project-default **relative** lethal
threshold (one lethal dose = the curve midpoint survival) to an **absolute**
survival target. First deferred-roadmap item of this session (the maintainer
approved "do all"). R-only, no engine change; the default behaviour stays
byte-identical.

## Model (the `q` shift)

The 4PL forward map is
`p(t, T) = low + (up - low) * plogis(-k * (log10(t) - mid))`,
`mid = log10(tref) - (T - CTmax) / z`. Define one lethal dose by the survival
target `target_surv` and let `q = qlogis((target_surv - low) / (up - low))`.

- **Lethal time to the target** (invert for `p = target_surv`):
  `log10(LT) = mid - q / k`, i.e.
  `LT(T) = tref * 10^((CTmax - T) / z - q / k)` -- exactly `derive_lt()` at
  survival `target_surv`. Damage rate `dmg(T) = 1 / LT(T)`.
- **Dose -> survival.** Under constant `T`, `D = t / LT`, so
  `log10(D) = log10(t) - mid + q / k`, hence the true 4PL survival is
  `survival(D) = low + (up - low) * plogis(-k * log10(D) + q)`. At `D = 1`:
  `survival = low + (up - low) * plogis(q) = target_surv` (one lethal dose
  reaches the target).

At `target_surv = (low + up) / 2` the ratio is `0.5` and `qlogis(0.5) = 0`
exactly, so both expressions collapse to the existing relative-threshold code:
the default path is unchanged. The change is the single additive `q` term in the
lethal-time exponent and the survival logit.

## Tests of the tests

The headline test pins the absolute target to the independently-implemented
`derive_lt()` (the Phase-4 4PL inverse): one lethal time of constant exposure to
the `target_surv` lethal time must give `dose -> 1` and `survival -> target_surv`.
The midpoint-equivalence test asserts `target_surv = (low + up) / 2` reproduces
the relative default to `1e-10` (the `q = 0` reduction). The validation test
pins the strict `(low, up)` range and the single-value requirement.
