# After Task: Random intercept on log_z (item 5, v0.3)

**Date:** 2026-06-17

**Task:** Implement a random intercept on `log_z`
(`log_z ~ <fixed> + (1 | group)`), the symmetric counterpart of the v0.2 random
intercept on `CTmax` — the first slice of roadmap item 5 ("random effects beyond a
single CTmax intercept"). Byte-identical gate on the engine; TDD; full capability
sync; team review.

## Created / Changed

- **Engine** `src/profile_tls.cpp`: added `DATA_IVECTOR(re_index_logz)`,
  `PARAMETER_VECTOR(b_logz)`, `PARAMETER(log_sd_logz)` (appended at the end of the
  declarations so the existing free-parameter order is untouched). In the loop the
  deviation is added to `logz_i` **before** `exp()`, guarded by `b_logz.size()>0`;
  a `dnorm(b_logz, 0, exp(log_sd_logz))` prior and `sigma_logz`/`b_logz`
  REPORT/ADREPORT are added under the same guard.
- **Parser** `R/formula.R`: `tls_extract_ct_re()` generalised to
  `tls_extract_re(rhs, data, param)` and called for both `CTmax` and `log_z`; the
  RE bar is stripped before the shared fixed-design comparison; a warning fires
  when the **same** grouping factor is on both sub-parameters (independent
  variances). Shape-RE rejection message and stale `@noRd` docblock updated.
- **Plumbing** `R/fit_tls.R`: builds both blocks; `random` names only the active
  ones; maps the absent block's `log_sd_*` to `factor(NA)`; stores `fit$re_logz`;
  adds the `sigma_logz` estimate row.
- **Helpers** `R/utils.R`: `tls_has_re()` and `tls_re_blocks()` — the single source
  of truth for per-block bookkeeping (chosen over a stored `re_blocks` field to keep
  the persisted object contract and back-compat).
- **Routing generalised** from `fit$re` to `tls_has_re()` / `tls_re_blocks()`:
  `R/methods.R` (`ranef()` iterates blocks, `print` second RE line),
  `R/extract.R` (`sigma_logz` Wald), `R/confint.R` (gates + Wald-only variance
  components), `R/profile.R` (contrast-under-RE gate + the contrast refit carries
  the new engine fields), `R/bootstrap.R` (redraws every active block; `sigma_logz`
  target scale), `R/plotting.R` (Confidence-Eye RE gate).
- **Simulation** `R/simulate.R`: `re_sd_z` mode (log-scale deviation on `z`),
  combinable with `re_sd`; preserves the exact RNG stream of an `re_sd`-only call.
- **Tests** `tests/testthat/test-random-effects-logz.R` (new, 13 `test_that`); two
  scope tests in `test-formula.R` / `test-random-effects.R` flipped from
  "log_z RE rejected" to "log_z RE accepted; shape RE rejected".
- **Docs synced (one commit):** `NEWS.md` (0.3.0), `ROADMAP.md`,
  `docs/design/08-random-effects.md`, `docs/dev-log/known-limitations.md`,

## Tests Of The Tests

The "no-RE fit unchanged" test asserts the formula fit equals the column fit
(tol 1e-8) and that `sigma_logz` is absent — the byte-identical guard at the R
level. The recovery test averages `sigma_logz` over 5 seeds (ML, mildly low). The
same-grouping test asserts BOTH the warning AND a successful two-block fit (so the
warning is not a hard error). `simulate_tls(re_sd_z=)` validation pins the RNG
back-compat contract.

## Team Learning

Pre-implementation review by the four engine/math/architecture/inference
perspectives, then a faithful mirror of an existing proven pattern, is the recipe
for a clean byte-identical engine change. Centralising RE bookkeeping in
`tls_re_blocks()` at N=2 blocks (rather than a third copy-paste) is the cheapest
point to abstract and sets up the shapes / crossed-factor slices.

## Known Limitations

`sigma_CTmax` / `sigma_logz` are ML variance components (biased low, few groups).
`sigma_logz` is a SD on `log(z)` (multiplicative on `z`). REs supported on `CTmax`
and `log_z` only (not shapes; no random slopes; one grouping factor each). The same
grouping on both fits **independent** variances (no correlation term) and warns; a
correlated `(CTmax, log_z)` RE and a `sigma_logz`/`phi` dispersion ridge are the
honest hazards — use `bayesTLS` for the correlated structure. See
`docs/design/08-random-effects.md`.
