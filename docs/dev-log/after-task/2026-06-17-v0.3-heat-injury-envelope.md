# After Task: Heat-injury bootstrap envelope + plot export (v0.3)

**Date:** 2026-06-17

**Task:** Promote the heat-injury bootstrap band to a real, exported capability,
removing the `profileTLS:::tls_bootstrap_replicates` reach into the package from
`vignettes/heat-injury.Rmd` (the fragility flagged in the v0.2.0 handover).

## Created / Changed

- **Refactor** `R/heat_injury.R`: extracted the forward-Euler dose-accumulation
  core of `predict_heat_injury()` into an internal `tls_injury_traj(low, up, k,
  CTmax, z, tref, time, temp, q, t_c, rep_rate, irreversible)`. `predict_heat_injury()`
  now calls it at the fitted parameters; the output is byte-identical (CTmax / z
  are constant within a group, so the scalar call reproduces the previous vectorised
  expression exactly).
- **New export** `heat_injury_envelope()`: redraws the curve parameters by
  parametric bootstrap (`tls_bootstrap_replicates()`, the same machinery
  `confint(method = "bootstrap")` uses), re-integrates each converged draw through
  `tls_injury_traj()`, and returns a tidy `time` / `temp` / `survival` /
  `conf.low` / `conf.high` compatibility band. Resolves shared / per-group
  replicate columns; supports `target_surv` (recomputing the dose→survival offset
  `q` per draw), `t_c`, `repair`, `irreversible`, `nboot`, `conf.level`, `seed`.
- **New export** `plot_heat_injury()`: draws the point trajectory inside the band
  with an honest caption that names the variability source and the replicate count
  ("pointwise parametric-bootstrap compatibility band (prior-free; curve
  parameters redrawn from the fitted model; B = <nboot>)"), never posterior /
  credible. `time_div` rescales the x-axis (e.g. hours→days).
- **Vignette** `vignettes/heat-injury.Rmd`: the bootstrap section now calls
  `heat_injury_envelope()` + `plot_heat_injury()`; the `:::` and the ~20-line
  inline `surv_traj` re-implementation are gone.
- **Tests** `tests/testthat/test-heat-injury-envelope.R` (new): band validity +
  point match (shared integrator) + reproducibility; envelope widens as survival
  falls; `plot_heat_injury()` returns a ggplot with a non-posterior caption;
  argument validation; an absolute-`target_surv` envelope.
- **Docs synced:** `NEWS.md` (0.3.0), `docs/design/46-capability-matrix.md`,
  regenerated `NAMESPACE` + `man/heat_injury_envelope.Rd` + `man/plot_heat_injury.Rd`.

## Outcomes

The `predict_heat_injury()` refactor is byte-identical (the existing
`test-heat-injury` context passed unchanged), and the new band reuses the same
integrator, so the band and the point trajectory are consistent by construction —
no second copy of the dose-accumulation map to drift. The vignette no longer
reaches into the package internals.

## Consistency Review

The honest-uncertainty language is uniform: "compatibility band, prior-free, never
a posterior / credible band" in the function docs, the plot caption, the vignette
prose, NEWS, and the capability matrix. No `:::` remains in `vignettes/`.

## Tests Of The Tests

The band-validity test asserts `conf.low <= conf.high`, the band lies in [0, 1],
and the `survival` column equals `predict_heat_injury()` to 1e-10 (proving the
shared integrator). The "widens" test guards the exponential-sensitivity story.
The caption test enforces the no-posterior contract.

## What Did Not Go Smoothly

The byte-identical integrator extraction went cleanly (passing the group's scalar
`CTmax`/`z`, constant within a group, reproduces the per-row arithmetic). Two
test/figure wrinkles, both caught and fixed before commit:
- The first "envelope widens" test asserted the band is widest at the END of the
  trace; that is physically wrong — the band is widest through the **transition**,
  and the original sinusoidal trace cooled at the end (the tail band was 0.02
  *narrower* than the start). Fixed with a heating-ramp trace and a robust
  `max(width) > 2 * width[1]` assertion.
- Florence's figure-gate review flagged that the caption named neither the
  variability source nor the replicate count; added both ("curve parameters
  redrawn from the fitted model; B = <nboot>").

## Team Learning

Extracting a shared scalar-parameter integrator is the clean way to add a
bootstrap envelope without re-implementing (and risking drift from) the point-
estimate map — the vignette's old inline `surv_traj` was exactly such a drift risk.
Bootstrap-band test assertions should target the high-damage / transition region
(where uncertainty peaks), not the saturated tail. The figure-gate (Florence)
caught a real honesty gap in the caption that a code-only review would have missed.

## Known Limitations

The envelope is a parametric bootstrap (redraws curve parameters, refits per
replicate); it is slow for large `nboot` and inherits the fit's stability. It is a
compatibility band, not a posterior. Repair remains a non-identified scenario layer.
