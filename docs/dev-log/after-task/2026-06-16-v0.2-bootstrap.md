# After-task: parametric bootstrap CIs + bootstrap fallback (v0.2)

## Task goal

Give profileTLS prior-free interval *parity with bayesTLS*: a likelihood-path way
to always return an interval, including the cases where the profile does not close
or the Hessian is not positive definite (`pdHess = FALSE`). Requested in-thread by
the maintainer ("you can do bootstrapping if profile does not go or pdHess = F";
"try to be as comparable as bayesTLS"). Design choices confirmed with the
maintainer: **auto + explicit** trigger, **parametric** bootstrap.

## What was implemented

- `R/bootstrap.R` (new):
  - `tls_bootstrap_replicates()` regenerates `y` at the **observed design** from
    the compiled model's `p_fitted` (drawn binomial or beta-binomial per the
    fit's family and `phi`, the R-PHI convention), refits `nboot` times through
    the retained `fit$tmb_inputs` warm-started at the MLE (the same refit path the
    profile uses), and collects natural-scale estimates. Non-converged replicates
    are recorded `NA` and excluded. Optional `seed` is set and restored locally.
  - `tls_boot_target()` maps a target name (direct params, grouped names, the
    `log_z` family, and `dCTmax`/`dz`/`dlog_z` contrasts) to its replicate vector
    **on the construction scale** (z/k/phi on log, low on logit, CTmax/up on
    identity) plus a monotone back-transform.
  - `tls_confint_bootstrap()` returns the standard 8-column confint tibble with
    `method = "bootstrap"`, `conf.status = "bootstrap"`; too few usable replicates
    -> `NA` + `conf.status = "bootstrap_unstable"` + a warning.
- `R/confint.R`: `method = "bootstrap"` plus a default `fallback = TRUE` that
  replaces any non-closing profile row (or every row when `pdHess = FALSE`) with a
  bootstrap interval, with a message; new args `fallback`, `nboot`, `boot_seed`.
- `R/plotting.R`: `plot_confidence_eye()` renders a `"bootstrap interval"` as a
  distinct (purple) lens and reports the source ("parametric bootstrap" or
  "profile likelihood (with fallback)") in the caption; new passthrough args
  `method = "bootstrap"`, `fallback`, `nboot`, `boot_seed`.

## Mathematical contract touched

The forward map and family draws are exactly those of `simulate_tls()` /
`src/profile_tls.cpp` (no new model). The interval construction reuses the profile
machinery's map-refit and `tls_mle_par_list()` warm start. Key invariant
preserved: **exact equivariance**. Because percentiles are taken on the
construction scale and back-transformed, the bootstrap `z` interval equals
`exp()` of the `log_z` interval to machine precision (verified: max abs diff 0).

## Tests of the tests

Deterministic `boot_seed` throughout. `test-bootstrap.R` checks: contract
(8-col shape, finite, `conf.low <= estimate <= conf.high`); RNG restoration
(`.Random.seed` unchanged after a seeded call); exact equivariance
`z == exp(log_z)` (tol 1e-10); truth coverage; auto-fallback yields
`method == "bootstrap"` with a message; `fallback = FALSE` keeps the open `NA`;
per-group grouped bootstrap with A < B ordering; beta-binomial draw path.

## What did not go smoothly

- The recon-written `data-raw/build_benchmark_cache.R` had two broken summarisers
  (guessed `extract_tdt`/`ts_ci` schemas); fixed against the real bayesTLS 1.0.0
  API before the MCMC run (separate task #1).
- First instinct (percentile on the natural scale) would have made `z` and
  `log_z` only approximately equivariant because `quantile()` interpolation does
  not commute with `log`. Fixed by taking percentiles on the construction scale.

## Known limitations and next actions

- Parametric bootstrap assumes the fitted model is correct (as the profile does);
  on genuinely non-identified parameters the interval is honestly wide, not magic.
- Nonparametric / cluster bootstrap is not implemented (parametric only, by
  design for the designed temp x duration grid).
- A user-facing `bootstrap_tls()` returning the full replicate distribution is a
  possible future addition (the internal engine already produces it).
- Next: wire the real cache + bootstrap into the comparison vignette and rebuild
  the site (tasks #3-#5), then the RE-1 random-intercept engine.
