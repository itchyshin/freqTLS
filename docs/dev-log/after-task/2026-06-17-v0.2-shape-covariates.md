# After-task: grouped covariate effects on low/up/log_k (v0.2, covariates 11b)

## Task goal

Expose the engine groundwork from 11a as a user capability: let `low`, `up`, and
`log_k` vary by a grouping factor through the formula interface, with per-group
estimates, Wald intervals, and prediction. Final feature of the maintainer-
approved ("do all") v0.2 run. Builds on the byte-identical shape-design engine
(commit `bee8d04`).

## Scope (recorded)

Delivered: grouped shapes (`low ~ group`, `up ~ group`, `log_k ~ group`) via the
formula interface, with per-group point estimates, **Wald (delta-method)**
intervals (`confint(method = "wald")` / `tidy_parameters()`), and per-group
`predict()` (and the plots built on it). `simulate_tls()` accepts per-group
`low` / `up` / `k` for recovery tests.

Deliberately out of scope (clear errors / documented): profile and bootstrap
intervals for grouped shape coordinates (`low:<g>` etc.) -- a clear error directs
to Wald; `CTmax` / `z` profiling and bootstrap are unaffected. General continuous
covariates on the shapes (the prediction resolver is group-keyed). The three
shapes must share one grouping factor, matching the `CTmax` / `log_z` factor.

## What changed

- `R/formula.R`: removed the v0.1 shape-restriction; `tls_parse_formula()` builds
  `X_low` / `X_gap` (from `up`) / `X_logk` via `tls_design_from_rhs`, enforces the
  shared-design and same-as-CTmax constraints, and returns the shape designs.
- `R/fit_tls.R`: the formula branch forwards the shape designs; `n_shape` flows to
  `tls_default_start()` and `tls_name_map()`; `tls_estimates()` emits per-group
  `low:<g>` / `up:<g>` / `k:<g>` rows with per-group SEs.
- `R/utils.R` (`tls_name_map`): per-group shape coordinates (`beta_low[j]` ->
  `low:<level>`, logit; `beta_logk[j]` -> `k:<level>`, log; `beta_gap[j]` -> "gap"
  surfaced as `up` separately), gated by `n_shape`.
- `R/extract.R` (`tls_wald_natural`): grouped `up:<g>` Wald via the per-column
  ADREPORT `up` SE; grouped `low` / `k` fall out of the existing link-endpoint
  loop.
- `R/predict.R`: `tls_predict_pars()` now resolves per-row `CTmax` / `z` / `low` /
  `up` / `k` (shared recycled, grouped indexed by the row's group); `predict()`,
  `derive_lt()`, `derive_ctmax()` use it (and `predict_heat_injury()` in
  `R/heat_injury.R`); the unused `tls_shape_estimates()` helper was removed.
- `R/profile.R`: a clear error for grouped shape-coordinate profile targets
  (use Wald). `R/bootstrap.R`: a clear error for grouped-shape fits (use Wald).
- `R/simulate.R`: `low` / `up` / `k` recycle per group like `CTmax` / `z`; the
  binomial / beta-binomial RNG path is unchanged (existing seeded tests
  byte-identical).

## Checks run and exact outcomes

- `devtools::document()` -> OK (no warnings).
- `devtools::test()` -> `PASS 379 | FAIL 0 | WARN 0 | SKIP 0` (was 368; +11 in
  `test-shape-covariates.R`; one existing v0.1 formula test updated to the v0.2
  constraint). All shared-shape paths unchanged.
- `rcmdcheck` (`_R_CHECK_FORCE_SUGGESTS_=false`) -> pending in the check-log.
- Behaviour: a 2-group fit with truth `k = c(4, 8)` recovers `k:A = 3.58`,
  `k:B = 7.66` with finite per-group SEs; `tidy_parameters()` gives `k:A`
  [2.93, 4.37], `k:B` [6.30, 9.32]; per-group `low` / `up` / `CTmax` / `z`
  recover; `predict()` returns per-group survival.

## Tests of the tests

`test-shape-covariates.R`: per-group `log_k` recovery (B steeper than A); a fully
grouped fit yields `low:<g>` / `up:<g>` / `k:<g>` rows + Wald CI + per-group
`predict`; and the shared-design constraint (a partial shape predictor errors).
An existing `test-formula.R` v0.1 test ("a predictor on a shape parameter is a
v0.1 error") was updated to assert the v0.2 constraint ("must use the same
predictor").

## What did not go smoothly

- A first run failed one existing formula test that encoded the v0.1 shape
  restriction; updated to the v0.2 semantics (shape predictors are now allowed,
  but the three must share one design).
- The `up` Wald path special-cased a single `up` row; generalised to the
  per-group `up:<g>` rows via the per-column ADREPORT `up` SE.

## Known limitations and next actions

- Grouped shape intervals are Wald only; profile / bootstrap for them is a future
  item, as is general continuous covariates on the shapes.
- This completes the v0.2 feature roadmap (beta, RE profiling, T_crit,
  heat-injury, covariate shapes). Next: polish (dashboard refresh, capability
  matrix, vignettes, site) and the frequentist-vs-Bayesian article (a standalone
  vignette also covering clamping / shrinkage / penalties as implicit priors,
  with references).
