# After Task: Landing-Page Function Map and CRAN Readiness

Date: 2026-07-11
Branch: `build/freqtls`
Reader: freqTLS contributors and a release reviewer

## 1. Goal

Repair the broken function map in the get-started article and determine whether
freqTLS 0.1.0 is ready for a first CRAN submission.

## 2. Implemented

The source SVG was valid. Pandoc parsed `get_*_summary()` and
`get_*_draws()` as Markdown emphasis when the vignette emitted the SVG as
ordinary `results = "asis"` output. The resulting HTML `<em>` element ended the
browser's SVG context, so the rest of the map flowed as ordinary page text.

`vignettes/freqTLS.Rmd` now wraps the SVG in a Pandoc raw-HTML fence. The page
also calls the diagram the core workflow rather than the whole API. The site
builder now checks the rendered artifact and fails if this corruption returns.

## 3a. Decisions and Rejected Alternatives

This task did not change the four-parameter logistic model, direct `CTmax`/`z`
parameterisation, asymptote bounds, likelihoods, interval algorithms, or
Confidence-Eye geometry. It changed only how an existing workflow diagram is
passed through Pandoc and how the rendered result is validated.

The chosen fix keeps the accessible inline SVG and wraps it in a Pandoc
raw-HTML fence. Escaping underscores and asterisks was rejected because Pandoc
can decode entities before applying emphasis. `knitr::include_graphics()` was
also rejected for this repair because the earlier inline approach was adopted
after pkgdown failed to copy the co-located SVG. The raw fence fixes the parser
boundary without adding asset-copy behavior or a raster duplicate.

## 8. Consistency Audit

The map no longer claims to show the whole exported API. The targeted scans are
recorded verbatim in the check log. They also confirmed a larger release-story
problem: `AGENTS.md` and `SPEC.md` still describe Beta responses, formula syntax,
random effects, heat-injury prediction, and CRAN hardening as v0.1 non-goals,
while code, README, NEWS, and parts of the capability matrix describe them as
implemented. Known limitations and the capability matrix contain internally
contradictory status statements. These were audited, not rewritten, because a
scope reconciliation is a separate release decision.

The deliberate `bayesTLS` posterior/credible comparisons remain distinct from
freqTLS confidence-interval language.

Memory receipt: `memory-recall` found no freqTLS-specific prior record, and the
repository router found no freqTLS LOAD-FIRST manifest. The repository's
`AGENTS.md`, `SPEC.md`, CRAN, R-package, figure, prose, and after-task skills
therefore shaped the work. Golden Set: not run because no registered cross-repo
mistake case covers Pandoc emphasis inside inline SVG; the pre-fix live DOM and
post-fix rendered DOM served as the deterministic failure/recovery test.

## 9. What Did Not Go Smoothly

`pkgdown::build_article()` initially failed because that narrow helper assumes
freqTLS is already installed. Installing the package into `/tmp/freqTLS-lib`
made the targeted build reproducible. The full site and CRAN checks were slow
because they refit TMB examples and rebuild every vignette, but they completed.

`devtools::document()` proved that two hand-synchronised Rd pages were not
byte-identical to roxygen output and exposed two pre-existing roxygen warnings.
The regenerated `tdt_unit_to_minutes.Rd` now makes the unresolved
`derive_tdt_curve` link visible to `R CMD check`.

## 10. Known Residuals

freqTLS is not ready for CRAN submission. The main release gates are:

1. Reconcile the authoritative v0.1 scope across `AGENTS.md`, `SPEC.md`, README,
   NEWS, the capability matrix, known limitations, and the shipped code.
2. Fix the unresolved `derive_tdt_curve` Rd link and the malformed `@noRd` tag.
3. Decide how to handle the non-mainstream `bayesTLS` Suggest and test with the
   final dependency policy.
4. Exclude or relocate `output/` and decide whether top-level `scripts/` ships;
   the current source tarball is about 12.6 MB.
5. Add first-submission administration: `cran-comments.md`, a copyright-holder
   role, final author consent, non-development NEWS wording, and CRAN install
   instructions.
6. Fix the canonical trailing-slash site URL, review DOI 403 results, reconcile
   bundled-data licensing inventories, and expand final CI beyond Ubuntu
   release.

The benchmark-sanity test also has a stale skip message even though the current
cache is built from bayesTLS raw datasets.

### Next Actions

Treat the CRAN work as a separate release-remediation slice. Start with the
scope decision and package-build exclusions, because they determine the public
story and tarball contents. Then fix documentation/metadata, rerun strict and
relaxed `--as-cran` checks, and add multi-platform evidence. Deploy the pkgdown
site only after reviewing and committing the landing-page repair.

## 11. Team Learning

Inline SVG emitted from an R Markdown chunk is not opaque merely because blank
lines are removed. Wildcard function names can still be parsed as Markdown.
Use a Pandoc raw-HTML fence and validate the generated HTML whenever SVG text
contains Markdown punctuation. This task used Ada/Emmy, Grace, and
Florence/Pat/Grace role perspectives; the browser and full-site checks supplied
the independent rendered evidence.
