# After-task: derive_tcrit() rate-multiplier critical temperature (v0.2)

## Task goal

Add `T_crit`, the rate-multiplier lethal critical temperature -- the last piece of
the `bayesTLS` `extract_tdt()` absolute family. Third of the maintainer-approved
("do all") v0.2 items; R-only.

## Definition (confirmed from the bayesTLS source, not assumed)

`R/extract_tdt.R` (line ~721): per draw, `T_crit = CTmax + z * log10(r*/100)`,
where `r*` is a damage-rate floor in **percent of the lethal dose per hour**,
sampled uniformly on `log10` over `TC_rate_range` (default `0.1`--`1`). It is a
deterministic transform of the per-draw `CTmax` and `z`; bayesTLS injects extra
uncertainty by sampling `r*` (a `runif`). It is documented and guarded as
**lethal-endpoint only** (sublethal `z` is steeper, driving `T_crit` implausibly
low).

## Design calls (recorded)

- profileTLS `derive_tcrit(object, rate = 1, group = NULL)` returns the
  deterministic transform of the fitted `CTmax`/`z`; `rate` is a **fixed input**,
  not sampled. The `runif` rate draw has no ML meaning, so it is deliberately not
  copied -- a fair novelty: profileTLS gives `T_crit(rate)` as a clean transform
  (uncertainty propagates from the `CTmax`/`z` compatibility intervals) where the
  Bayesian path only gets a posterior by mixing in the rate draw.
- Mirrors the `derive_ctmax()` API exactly (single `group` for grouped fits, a
  vectorised quantity argument, a numeric return). `rate` is vectorised.
- Emits the lethal-endpoint caveat once per call (the package's "never silent"
  validity-guard ethos; suppressible).
