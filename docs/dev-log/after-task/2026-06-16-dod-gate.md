## What did not go smoothly

- `check_tls()` takes a fitted object (warnings 1-8 re-run post-fit), not raw
  data; the raw-data path is the internal `check_tls_data()` invoked by
  `fit_tls()`. NEWS.md:39 ("surfaced by `check_tls()` and during fitting and
  profiling") is accurate but the README/NEWS phrasing could mislead a reader into
  calling `check_tls(data, y=, n=, ...)`. Non-blocking (see improvements).
- `simulate_tls()` correctly rejects `low = up`, so degenerate no/all-mortality
  data must be built by hand -- a good guardrail, noted so future auditors do not
  mistake it for a bug.
