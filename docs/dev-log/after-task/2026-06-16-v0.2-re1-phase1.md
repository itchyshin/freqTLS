# After-task: RE-1 Phase 1 — random intercept on CTmax (v0.2)

## Task goal

Add the gated v0.2 milestone: a single random intercept on `CTmax`
(`CTmax ~ <fixed> + (1 | group)`) by TMB Laplace, matching the bayesTLS
RE-on-the-midpoint configuration, with the **no-RE path byte-identical** to the
fixed-effects model (the hard gate). Autonomous overnight work toward the
finish-the-package goal.

## What was implemented

- **Engine** (`src/profile_tls.cpp`): `DATA_IVECTOR(re_index)`,
  `PARAMETER_VECTOR(b_CT)`, `PARAMETER(log_sd_CT)`. Per observation,
  `CT_i += b_CT(re_index(i))` and a `dnorm(b_CT, 0, exp(log_sd_CT))` prior are
  added **only when `b_CT.size() > 0`**; `sigma_CT` and `b_CT` are
  REPORT/ADREPORTed only then. No-RE => empty `b_CT`, mapped `log_sd_CT`,
  `random = NULL`, so the nll, free parameters, optimiser path, and reports are
  unchanged.
- **fit_tls / engine**: `fit_tls_engine()` gains `random`; `fit_tls()` builds
  `re_index`/`b_CT`/`log_sd_CT`, sets `random = "b_CT"` for RE, stores `fit$re`
  and `fit$tmb_inputs$random`, and surfaces `sigma_CTmax` (estimate + sdreport
  SE) for RE fits. The contrast refit (`profile.R`) passes the no-RE RE fields.
- **Formula** (`R/formula.R`): `tls_extract_ct_re()` splits a single
  `(1 | group)` off the `CTmax` RHS, validates the scope, and returns the `re`
  spec; non-RE formulas are unchanged.
- **simulate_tls**: `re_sd` / `n_re_groups` / `re_group_name` RE mode.
- **Tests**: `tests/testthat/test-random-effects.R`.
- **Docs**: `docs/design/08-random-effects.md`; capability matrix, NEWS,
  ROADMAP, known-limitations synced; `tls_bf` / `simulate_tls` roxygen.

## Mathematical contract touched

The 4PL forward map is unchanged; the only addition is the group deviation on the
`CTmax` coordinate, `CTmax_i = X_CT beta_CT + b_{g(i)}`, with `b_g ~ N(0,
sigma_CTmax^2)` integrated out by Laplace. `sigma_CTmax = exp(log_sd_CT)`. The
direct-`CTmax`/`z` parameterisation and nested-gap asymptotes are untouched.

## Checks run and exact outcomes

- `pkgload::load_all()` recompiled the cpp; full suite **251 pass / 0 fail**
  after the recompile — the byte-identical gate held before any RE-request path
  existed.
- RE probe (binomial, `re_sd = 1.5`, 25 groups, seed 42): converged
  (code 0, pdHess TRUE), CTmax 36.26, z 3.99, sigma_CTmax 1.92.
- Recovery across 8 sims (30 groups): CTmax mean 36.015, z mean 4.001 (unbiased);
  sigma_CTmax mean 1.277 for true 1.5 — the expected ML downward bias.
- Full suite after RE tests + fixes: **280 pass / 0 fail / 0 warn** (local; the
  averaged recovery test ran, not skipped).
- `R CMD check (_R_CHECK_FORCE_SUGGESTS_=false)`: see check-log entry.

## Tests of the tests

Two real failures surfaced and were fixed, not papered over:
1. `obj$report()` (no-arg) reads `last.par`, which `sdreport()` can leave
   off-optimum; this perturbed the grouped fit's estimates and broke the tight
   `dlog_z` contrast test. Fixed by re-pinning `obj$fn(opt$par)` before
   `report()` (deterministic; byte-identical for no-RE, and re-solves the inner
   problem for RE). The 251 gate confirmed the restoration.
2. The old "RE on CTmax is a deferred error" test was now obsolete; repurposed to
   assert the parser accepts `(1 | group)` on CTmax and still rejects RE on
   `log_z`.

## Consistency audit

`fit$re`, `tmb_inputs$random`, the cpp guards, and the formula scope errors are
aligned. `sigma_CTmax` naming is stable across the cpp ADREPORT (`sigma_CT`), the
estimates table, and the docs. Compatibility language preserved.

## What did not go smoothly

## Known limitations and next actions

- `sigma_CTmax` is ML (biased low with few groups; no REML). Documented.
- RE Phase 2 (next): profile / `confint` for `sigma_CTmax` and `CTmax`/`z` under
  the RE (re-running Laplace with `random = "b_CT"` at each profile point), and
  `ranef()` BLUPs. A flagship RE example for the README / a vignette.
- RE + bootstrap (`confint(method = "bootstrap")` on an RE fit) is not yet wired
  (the bootstrap refit path assumes no random block); guard or extend in Phase 2.
