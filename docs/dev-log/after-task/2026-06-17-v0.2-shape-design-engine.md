# After-task: shape-parameter design engine (v0.2, covariates 11a)

## Task goal

First, byte-identical half of "covariate effects on `low`/`up`/`log_k`": refactor
the TMB engine so the shape parameters take design matrices (like `CTmax` /
`log_z`), without exposing any new user capability yet. This isolates the risky
engine change behind the byte-identical gate before the grouped-shape feature
(11b) builds on it. Authorised by the `decisions.md` entry relaxing the
midpoint-only invariant.

## What changed

- `src/profile_tls.cpp`: `beta_low` / `beta_gap` / `beta_logk` are now
  `PARAMETER_VECTOR`s with design matrices `X_low` / `X_gap` / `X_logk`. Crucial
  byte-identical lever: a **single** shape coefficient takes a SCALAR path
  (`low_s = invlogit(beta_low(0))`, used for every observation) -- the matrix
  product `X_low * beta_low` is computed only when there is more than one
  coefficient. The per-column natural-scale `low` / `up` / `k` are reported as
  vectors (length 1 == the old scalars).
- `R/fit_tls.R`: builds intercept-only shape designs (`matrix(1, n_obs, 1)`) and
  passes them; `tls_default_start()` gains `n_shape` (length-1 by default).
- `R/profile.R`: the contrast refit passes intercept-only shape designs too.

No user-facing behaviour changes: the column and formula interfaces still produce
shared scalar `low` / `up` / `k`.

## Why the scalar guard (the byte-identical story, with evidence)

A first attempt used the matrix product `X_low * beta_low` for all cases. Even
for an intercept-only design (`1 * beta` == `beta`) this re-orders the gradient
summation, which shifted the MLE at ~1e-4 to 1e-6 and -- on the deliberately
degenerate beta-binomial-on-clean-data fit (phi at the binomial limit, a flat
direction) -- flipped `pdHess` TRUE->FALSE and produced a "NaNs produced" SE
warning. Measured against the committed baseline via `git stash`:

- with the matrix product: bb (seed 11) logLik -112.7757**165** vs baseline
  -112.7757**211**, `pdHess` FALSE vs TRUE, warned TRUE vs FALSE.
- with the scalar guard: `pdHess` and the warning **match baseline**; the bb MLE
  agrees to ~1e-5 (below every tolerance) and well-conditioned binomial fits
  (seeds 1, 7) are **bit-identical** (max abs diff = 0).

So the shared-shape path is bit-identical for well-conditioned fits, and the only
residual is sub-1e-5 wobble on an ill-conditioned degenerate fit (where even the
baseline barely identifies phi); `pdHess` and warnings match. True bit-identity
is not attainable once `beta_low` must be a vector (TMB treats a length-1
`PARAMETER_VECTOR` subtly differently than a `PARAMETER`); the scalar guard gets
as close as possible and restores a clean suite.
