## Compute decision

No new simulation campaign is required for the current release slice: the
tested estimands and simulation evidence already exist, and CRAN readiness is
currently a packaging, licensing, documentation, and platform problem. Local
macOS runs the document/test/check/pkgdown loop; GitHub Actions covers Ubuntu R
release/devel, Windows release, and macOS release; win-builder R-devel and R-hub
Linux/clang check the exact tarball independently.

If a gate exposes a statistical-evidence gap, Curie and Gauss will first define
the estimand, data-generating process, parameter grid, repetitions, seed policy,
stopping rule, and acceptance thresholds. Totoro is the quick rehearsal host
with each worker single-threaded and conservative shared-machine usage; the
claim-bearing rerun moves to DRAC with scheduler, image/module, job-array, wall
time, memory, and output provenance recorded before launch. Totoro and DRAC
denominators must not be mixed silently.

## Data redistribution gate

raw CSVs, and case-study source are preserved under the build-excluded
`data-raw/licensing-pending/snowgum/` tree. Snow-gum rows and metadata were
removed from the installed benchmark cache. The Kristineberg station extract,
which lacks explicit redistribution terms, is likewise preserved under
`data-raw/licensing-pending/kristineberg/`. Two unused environmental traces
derived from Open-Meteo/ERA5 and an Orsted/NicheMapR/NCEP workflow also lack a
complete recorded redistribution chain for every underlying provider; they are
preserved under `data-raw/licensing-pending/environmental-traces/` and have no
installed consumer. The package can proceed without these assets after the
final tarball inventory confirms their exclusion.
